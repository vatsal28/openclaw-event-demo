Practical tips

The stuff that actually made this usable.

Boot context“Start by reading identity, user profile, today’s notes, and project context.”Load the right context before the first answer.
Model routing“Use fast models by default; escalate only for writing, code, and hard debugging.”Spend reasoning power where it matters.
Subagents“For work over 2 minutes, spawn a worker and keep chat responsive.”Long edits and research move out of the main chat.
Distillation“Log daily notes, then promote only durable facts to long-term memory.”Useful memory beats giant memory.
Separate workspaces“Keep each user and project in separate files, memory, and boundaries.”Privacy and relevance stay clean.
Local LLMs“Use local models for cheap checks, private drafts, and cloud fallback.”Lower cost, better resilience.
Mini RCA“After every fix: what broke, why, what changed, prevention.”Turn failures into reusable lessons.
Humanizer skill“Before publishing, remove AI-slop and rewrite in my voice.”Posts, emails, and drafts get a voice pass.
